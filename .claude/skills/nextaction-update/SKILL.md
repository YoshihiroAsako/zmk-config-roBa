# nextaction-update

正本は各リポジトリの `AGENTS.md` §1(NextAction.md の運用)。矛盾したらそちらが優先する。このファイルは §1 の手順をチェックリスト化したもので、条文の複製ではない。

## 手順

1. 現在の `NextAction.md` を読む。
2. 直前の作業の完了・決定・承認・却下のみを、現在地 / 決定済み事項 / Next Actions / 保留・未決 の各セクションへ反映する。**却下された作業は「保留・未決」へ移す。** 指示で言及されていない項目は変更しない。
3. 「決定済み事項」「保留・未決」が**追記のみ**になっているか自己検証する(過去の記録を書き換えていないか)。「現在地」「Next Actions」は現状を反映して書き直してよい。
4. クローズ済みの項目(完了した Next Action、決着した保留事項)を、リポジトリ固有の履歴ファイル(通常 `Archive.md`)へ**末尾追記**で移す。見出し形式は `## YYYY-MM-DD: (件名)` + 本文。既存エントリの書き換え・並べ替え・削除はしない。
5. 変更差分をチャットで報告する。
