# キーマップ変更をキーボードへ反映する手順

このドキュメントは、Cursor上で変更したroBaのキーマッピング関連設定を、実際のキーボードへ反映するための手順メモです。

## 全体の流れ

Cursorで `config/roBa.keymap` などを編集しただけでは、キーボード本体にはまだ反映されません。

基本的な流れは次のとおりです。

1. Cursorでキーマップを編集する
2. 必要に応じて keymap-drawer 用のファイルも更新する
3. GitHubへcommit / pushする
4. GitHub Actionsでファームウェアをビルドする
5. 生成された `.uf2` ファイルをダウンロードする
6. キーボードをブートローダーモードにする
7. 対応する `.uf2` ファイルをキーボードへコピーする

## 変更する主なファイル

キーマップ本体は次のファイルです。

```text
config/roBa.keymap
```

表示用のキーマップ図を更新する場合は、次のファイルも関係します。

```text
keymap-drawer/roBa.yaml
keymap-drawer/roBa.svg
```

`keymap-drawer/roBa.yaml` や `keymap-drawer/roBa.svg` は、見た目の確認やドキュメント用途のファイルです。ファームウェアとしてキーボードに書き込まれる中心は `config/roBa.keymap` です。

## GitHub Actionsでビルドする

変更をGitHubへpushすると、GitHub Actionsが自動でZMKファームウェアをビルドします。

例:

```powershell
git status
git add config/roBa.keymap keymap-drawer/roBa.yaml keymap-drawer/roBa.svg
git commit -m "Update keymap"
git push
```

push後、GitHubのリポジトリ画面で `Actions` タブを開きます。新しいワークフロー実行が表示され、成功するとArtifactsからビルド済みファームウェアをダウンロードできます。

このリポジトリでは、主に次のファームウェアが生成されます。

```text
roBa_L.uf2
roBa_R.uf2
settings_reset.uf2
```

ファイル名はArtifactsの展開方法やビルド設定によって多少異なる場合がありますが、左手側用が `roBa_L`、右手側用が `roBa_R` です。

## どのUF2をどちらに書き込むか

roBaは左右分割キーボードなので、左右それぞれに対応したファームウェアを書き込みます。

```text
左手側: roBa_L の .uf2
右手側: roBa_R の .uf2
```

左右を逆に書き込むと、分割キーボードとしての役割やピン設定が合わなくなる可能性があります。特にroBaでは、物理的な左右の役割が異なります。

- `roBa_L`: 左手側、エンコーダーがある側
- `roBa_R`: 右手側、PMW3610トラックボールとSPI配線がある側

キーマップ変更だけの場合でも、基本的には左右両方に新しいファームウェアを書き込むのが安全です。

## ブートローダーモードに入る

UF2を書き込むには、対象のキーボード半分をブートローダーモードに入れます。

roBaは `seeeduino_xiao_ble` を使っているため、一般的にはリセットボタンを素早く2回押す「ダブルリセット」でブートローダーモードに入ります。

手順:

1. 書き込みたい側のキーボードをUSBケーブルでPCに接続する
2. その側のリセットボタンを素早く2回押す
3. Windowsに新しいUSBドライブとして表示されることを確認する

ドライブ名は環境によって異なります。たとえば次のような名前で表示されることがあります。

```text
XIAO-SENSE
XIAO-BLE
UF2BOOT
ZMK
NICENANO
```

名前よりも、「USBメモリのような新しいドライブがエクスプローラーに現れる」ことが目印です。

## roBaのリセットボタンを探す

roBa固有のケース上の位置は、ケースや組み立て状態によって異なる可能性があります。

まず探す場所は、各半分に載っているXIAO BLE基板上の小さいタクトスイッチです。

見分けるポイント:

- USB-C端子の近くにある小さいボタン
- 基板に `RST` または `RESET` と印字されていることがある
- 左右それぞれの半分にある
- ケースに入っている場合、底面や側面の小さい穴からピンなどで押す構造になっていることがある

roBaのPCB側に別途リセットスイッチが実装されている場合は、XIAO BLE基板上のボタンではなく、ケース外から押せる小さいボタンや穴がリセットボタンになっていることがあります。どちらの場合も、XIAO BLEのRESETにつながっているボタンであれば目的は同じです。

## `.uf2` をコピーするとは何か

ここでいう「コピーする」とは、ダウンロードした `.uf2` ファイルを、ブートローダーモードで表示されたUSBドライブへドラッグ＆ドロップする操作です。

左手側の例:

1. GitHub ActionsのArtifactsから `roBa_L` の `.uf2` をダウンロードする
2. 左手側をUSB接続する
3. 左手側をブートローダーモードにする
4. Windowsのエクスプローラーに表示された新しいUSBドライブを開く
5. `roBa_L.uf2` をそのドライブへドラッグ＆ドロップする
6. コピー完了後、ドライブが自動的に消えてキーボードが再起動する

右手側の例:

1. GitHub ActionsのArtifactsから `roBa_R` の `.uf2` をダウンロードする
2. 右手側をUSB接続する
3. 右手側をブートローダーモードにする
4. Windowsのエクスプローラーに表示された新しいUSBドライブを開く
5. `roBa_R.uf2` をそのドライブへドラッグ＆ドロップする
6. コピー完了後、ドライブが自動的に消えてキーボードが再起動する

コピー後にドライブが消えるのは正常です。UF2の書き込みが完了し、キーボードが通常モードで再起動したことを示します。

## `current.uf2` とコピー時エラーについて

ブートローダーモードで表示されたドライブに `current.uf2` が見えることがあります。これは現在書き込まれているファームウェアを示すための特殊なファイルで、通常は削除・編集しません。

ファームウェア更新時は、`current.uf2` はそのままにして、GitHub Actions で生成された `roBa_L` または `roBa_R` の `.uf2` をそのドライブへコピーします。コピーしたファイル名がドライブ内に残らなくても問題ありません。

Windows では UF2 コピー時に、次のようなエラーが表示されることがあります。

```text
0x800701B1: 存在しないデバイスを指定しました。
```

コピーの途中または直後にボードが自動再起動して bootloader ドライブが消えると、Windows がこのエラーを表示することがあります。エラー表示だけでは失敗とは断定できません。

次の状態なら、書き込みに成功している可能性があります。

- コピー後に bootloader ドライブが消えた
- USBを抜き差しするとキーボードとして復帰する
- 新しいキーマップの動作が反映されている

逆に、新しい動作が反映されていない場合は、もう一度 bootloader モードに入れて対応する `.uf2` をコピーします。USBハブ経由で不安定な場合は、PC本体の別USBポートや別ケーブルも試します。

## settings_resetについて

`settings_reset.uf2` は、Bluetoothペアリング情報などの永続設定を消すための特別なファームウェアです。

通常のキーマップ変更では使いません。

左右の接続やBluetoothペアリングが壊れた場合などに、必要に応じて次の流れで使います。

1. 左右両方に `settings_reset.uf2` を書き込む
2. その後、左手側に `roBa_L.uf2` を書き込む
3. 右手側に `roBa_R.uf2` を書き込む
4. PCやスマートフォン側のBluetooth登録を削除して、必要に応じて再ペアリングする

`settings_reset.uf2` を書き込んだだけでは通常利用には戻りません。必ず通常の `roBa_L` / `roBa_R` ファームウェアを書き込み直します。

## 反映されないときの確認ポイント

- GitHub Actionsのビルドが成功しているか
- 左手側に `roBa_L`、右手側に `roBa_R` を書き込んだか
- `.uf2` をブートローダードライブの直下にコピーしたか
- コピー後にドライブが自動で消えて再起動したか
- Bluetooth接続の場合、古いペアリング情報が残っていないか
- 片側だけでなく左右両方を書き換えたか

まずはUSB接続で左手側が期待どおり入力できるか確認すると、問題の切り分けがしやすくなります。
