# Excel ファイルの生成(xlsx-create)

openpyxl を使って .xlsx を生成するスキル。仕様の正本は共通 design 配下にある:

- 層1(全 Excel 共通の規律): `../ai-workspace-rules/design/xlsx/spec.md`
- 層2(事業計画 PL のシート構成の型): `../ai-workspace-rules/design/xlsx/pl-spec.md`
- 体裁(フォント・配色・命名): `../ai-workspace-rules/design/office-style.md`

設計の経緯: `ai-workspace-rules/docs/design/xlsx-pl-format-standardization.md`

## 前提

- 生成には Python(openpyxl)を使う。
- **生成前に層1 `spec.md` を読む。** 事業計画・収支計画の PL を作る場合は層2 `pl-spec.md` も読む。
- スタイルの優先順位: ① リポジトリ固有のブランド設定(`assets/brand/README.md` /
  `assets/brand/template.xlsx` 等が配置されていればそれ)→ ② 共通仕様(`design/xlsx/spec.md` +
  `design/office-style.md`)。
- 出力先はリポジトリの慣行に従う(`output/` または `deliverables/`)。慣行が無ければユーザーに確認する。
- 生成スクリプトは成果物と対で残す(層1 §6)。バイナリは Git 管理外とし、スクリプトから再生成できる状態にする。

## 手順

1. 目的(事業計画 PL / 収支計画 / 価格シミュレーション / 一覧表 等)、必要なシート構成、前提となる
   数値の出所(リポジトリ内の参照ドキュメント)をユーザーに確認する。既に指示に含まれていれば聞き直さない。
2. シート構成と主要な計算ロジック(何を入力に何を算出するか)を先にチャットで提示し、合意を得てから生成する。
   単純な一覧表・整形のみの場合はこのステップを省略してよい。
   - **事業計画 PL の場合は `pl-spec.md` §1 のシート構成(`0_前提`〜`10_定義`)を既定案として提示する。**
     必須シートは `0_前提` / `1_ドライバー` / `2_売上` / `3_売上原価` / `4_販管費` / `5_PL` /
     `7_シナリオ` / `8_ユニット`。
3. 生成スクリプトを書き、`<出力先>/<資料名>_<YYYYMMDD>.xlsx`(リポジトリの命名慣行があればそれ)に保存する。
   参照実装として `../ai-workspace-rules/design/xlsx/generate_template.py` を流用してよい
   (シナリオ切替の CHOOSE、複合グラフ、データラベルのセル参照注入まで実装済み)。
4. 生成後、openpyxl で開き直して次を確認し、結果を報告する:
   - シート名の一覧が設計どおりか
   - 数式セルが数式として入っているか(`data_only=False` で `=` 始まりを確認)
   - 代表的な計算結果の検算(数式と同じ計算を Python 側で行い一致を確認)
5. 規約チェックを実行して結果を報告する:

   ```bash
   python ../ai-workspace-rules/design/xlsx/validate.py <出力ファイル>.xlsx          # 通常
   python ../ai-workspace-rules/design/xlsx/validate.py <出力ファイル>.xlsx --pl     # 事業計画 PL
   ```

## 設計の規律(要点。詳細は spec.md)

- **計算結果の値を直書きしない。** 算出値はすべて Excel 数式(セル参照)で組み、ユーザーが前提を変えたら
  再計算されるようにする。
- 前提条件(単価・成長率・コンバージョン率などの入力値)は「前提」シートに分離し、計算シートからは
  セル参照で使う。同じ値を複数箇所に直書きしない。
- 前提セルには出所(どのドキュメント・どの決定に基づくか)を隣接セルに記す。根拠のない数値は `(仮)`、
  外部確認が必要なものは `[要確認]` を明記する。
- 月次・年次などの時系列は列方向に展開し、行に科目を置く。
- 数値セルには表示形式(桁区切り・%・通貨)を設定する。入力セルは `FBE5D6` で塗り、計算セルは無色にする。
- ウィンドウ枠を固定する。ただし**チャートを載せるシートは固定しない**(チャートが隠れるため)。

## 注意

- 機密数値を含んでも処理はローカルで完結するため使用可。外部サービスへの送信はしない。
  ただし共通 design 配下のテンプレートには機密数値を入れない(ダミー値で構成する)。
- 既存の .xlsx を上書きしない。同名ファイルがある場合は日付やサフィックスを変えて保存する。
- openpyxl は既存ファイルのグラフ・ピボットを保持できないことがあるため、ユーザー提供の Excel を
  編集する場合は事前にその旨を伝え、コピーに対して作業する。
- チャートの書式を Excel 上で手動変更しない(再生成で失われる)。変更は生成スクリプト側に加える。
- コミットはユーザー指示時のみ(AGENTS.md §6)。
