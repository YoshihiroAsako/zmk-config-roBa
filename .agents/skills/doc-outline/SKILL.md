# doc-outline

資料(スライド・文書)の構成・内容を検討する。設計根拠は `ai-workspace-rules/docs/design/skill-facilitation-modes.md`(起動・配布の改訂は `docs/design/skill-cross-agent-hybrid-invoke.md`)。

## 前提

- 起動条件(いずれか):
  - 定型句「構成検討」「資料の骨子」または `/doc-outline` の明示指示。この場合は必ず本スキルに従う。
  - スライド・文書の読者/目的/ストーリー/章立て/キーメッセージを決めたい相談で、まだファイル生成が主目的でないもの。
- 起動しない場合:
  - 「スライドを作って」「pptx を生成」など、生成そのものが主目的のとき → 既存の生成手順(pptx 手順・`design/pptx/`)へ。ただし構成が未確定なら本スキルの利用を提案してよい。
  - 方針の開いた検討が主のとき → `brainstorm` へ。
- このスキル内ではスライド・文書ファイルを生成しない。生成手順は既存の pptx 手順・`design/pptx/` を参照する。
- 構成案は初版はチャットのみで出力する。ファイル保存はユーザーが明示指示した場合のみ行い、その時点で `AGENTS.md` §2 判定・変更対象ファイル宣言を行う。
- 作業完了時・セッション終了時の定期・自発呼び出しは禁止。

## 進め方

1. 読者・目的・意思決定ポイント: 誰に何を伝え、読んだ相手にどう動いてほしいかを確認する。
2. ストーリー: 結論に至る流れ(課題→根拠→結論など)を整理する。
3. ページ/章立て案: 構成案をページ・章単位で提示する。
4. 各ページのキーメッセージ: ページごとに伝えたいことを1行で示す。

## 出力

チャットで次の固定見出しを使って報告する:

- 読者と目的
- ストーリー
- 構成案
- 各ページのキーメッセージ
- 未決

## 終了時の分岐

区切りのよいところで、次のいずれかを提示してユーザーに選んでもらう:

- 構成の改訂
- 生成フェーズへ(スライド/文書ファイルの作成)
- 壁打ち(`brainstorm`)に戻る
- 保留

生成フェーズへ進む場合は、このスキルの中では生成せず、既存の pptx 手順・`design/pptx/` 等の別スキル・既存手順へ渡す。生成に着手する前に、変更対象ファイルの宣言と `AGENTS.md` §2 判定を行う。
