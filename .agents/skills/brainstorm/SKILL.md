# brainstorm

開いた検討を進めるための会話ファシリテーション。設計根拠は `ai-workspace-rules/docs/design/skill-facilitation-modes.md`(起動・配布の改訂は `docs/design/skill-cross-agent-hybrid-invoke.md`)。

## 前提

- 起動条件(いずれか):
  - 定型句「壁打ち」「壁打ちモード」または `/brainstorm` の明示指示。この場合は必ず本スキルに従う。
  - 方針・選択肢・進め方が未確定で、整理・比較・仮結論を求める相談(実装やファイル作成が主目的でないもの)。
- 起動しない場合:
  - 具体的なコード修正・バグ調査・実装手順の実行が主目的のとき。
  - 設計ドキュメントや資料ファイルの作成そのものが主目的のとき(必要なら終了時の分岐から別作業へ)。
  - 「検討したい」単独など、対象が実装か壁打ちか判別できないとき → 短く確認するか通常対応とし、無理に本スキルへ固定しない。
- このスキル内では実装・ファイル変更・`NextAction.md` の更新を行わない。**設計ドキュメントの作成もこのスキルの中では行わない。**
- いきなり成果物(コード・資料・ドキュメント)を生成しない。
- 作業完了時・セッション終了時の定期・自発呼び出しは禁止。

## 進め方

1. 目的の確認: 何を決めたいか、何のための検討かを確認する。
2. 前提の洗い出し: 制約・既知の事実・関係者を整理する。
3. 選択肢を並べる: 案を複数提示する(1案しかない場合もその旨を明記する)。
4. 評価軸: 案を比較する観点を示す。
5. 仮結論と未決: 現時点の仮結論と、まだ決まっていない点を分けて示す。

## 出力

チャットで次の固定見出しを使って報告する:

- 目的
- 前提
- 選択肢
- 仮結論
- 未決
- 次の一手

## 終了時の分岐

区切りのよいところで、次のいずれかを提示してユーザーに選んでもらう:

- 深掘り継続
- `docs/design/` へ設計ドキュメント化
- 資料構成検討(`doc-outline`)へ渡す
- 保留

`AGENTS.md` §2(設計ドキュメント分岐)に該当しそうな内容が見えた場合は、その時点で「設計ドキュメント化」を提案してこのスキルを終了する。ユーザーが明示的に指示した場合のみ、§2 の手順に従って**別作業として** `docs/design/` にドキュメントを作成する。
